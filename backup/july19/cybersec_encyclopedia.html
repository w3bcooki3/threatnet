SOC Operations

Advanced SIEM Tuning & Optimization

Correlation Rule Development

Develop sophisticated correlation rules based on behavioral patterns rather than simple signature matching. Focus on temporal relationships and statistical anomalies.

# Advanced Correlation Rule Example (Splunk) index=security sourcetype=windows:security | eval hour=strftime(_time, "%H") | stats count by user, hour | eventstats avg(count) as avg_count, stdev(count) as std_count by user | eval zscore = (count - avg_count) / std_count | where zscore > 2 | where hour >= 22 OR hour <= 6

Behavioral Context Analysis

Correlate alerts with user behavior analytics (UBA) and entity behavior analytics (EBA) to identify truly anomalous activities.

# UBA Correlation Query SELECT u.username, a.alert_type, a.severity, LAG(a.timestamp) OVER (PARTITION BY u.username ORDER BY a.timestamp) as prev_alert, COUNT(*) OVER (PARTITION BY u.username, DATE(a.timestamp)) as daily_alerts FROM alerts a JOIN users u ON a.user_id = u.id WHERE a.timestamp >= NOW() - INTERVAL '7 days' ORDER BY a.timestamp DESC;

Threat Hunting

Behavioral Analysis & Anomaly Detection

Process Execution Analysis

Hunt for suspicious process execution patterns, including parent-child relationships, timing anomalies, and execution context.

# Hunt for Suspicious Process Execution DeviceProcessEvents | where ProcessCommandLine contains "powershell" and ProcessCommandLine contains "-enc" | extend DecodedCommand = base64_decode_tostring(extract(@"-enc\s+([A-Za-z0-9+/=]+)", 1, ProcessCommandLine)) | where DecodedCommand contains "DownloadString" or DecodedCommand contains "IEX" | project TimeGenerated, DeviceName, ProcessCommandLine, DecodedCommand

Network Communication Patterns

Identify command and control communications through beacon analysis, DNS patterns, and protocol anomalies.

Hunt Hypothesis: C2 beacons often exhibit regular timing patterns. Look for connections with consistent intervals (¬±10% jitter).

Lateral Movement Detection

# Lateral Movement Hunt SecurityEvent | where EventID in (4624, 4625, 4648) | where LogonType in (3, 9, 10) | summarize LoginCount = count(), DistinctComputers = dcount(Computer), DistinctIPs = dcount(IpAddress) by Account, bin(TimeGenerated, 1h) | where DistinctComputers > 5 or DistinctIPs > 3

Correlate vulnerability disclosures with threat actor exploitation patterns to prioritize patching efforts.

# Vulnerability-Threat Correlation SELECT v.cve_id, v.cvss_score, v.publish_date, COUNT(t.campaign_id) as exploitation_count, MIN(t.first_seen) as first_exploitation FROM vulnerabilities v LEFT JOIN threat_campaigns t ON v.cve_id = t.exploited_cve WHERE v.publish_date >= '2024-01-01' GROUP BY v.cve_id, v.cvss_score, v.publish_date ORDER BY exploitation_count DESC, v.cvss_score DESC;
