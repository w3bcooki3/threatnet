Network Reconnaissance Techniques

ThreatNet Team July 23, 2025 Network, Reconnaissance, Scanning

Network reconnaissance is the process of gathering information about a target network before launching an attack or conducting an assessment. It's a critical first step in both offensive and defensive cybersecurity.

Passive Reconnaissance

Gathering information without directly interacting with the target network. This minimizes detection risk.

  • WHOIS Lookups: Determine domain registration details, IP ranges, and contact information.
  • DNS Lookups: Use tools like `dig`, `nslookup`, or online DNS tools to find A, MX, NS, CNAME, and SPF records.
  • Google Hacking / Dorking: Use search engines to find publicly exposed files, directories, or vulnerable systems related to the target.
  • Shodan/Censys: Search for internet-connected devices, open ports, banners, and vulnerabilities.
  • Wayback Machine: Access historical versions of websites for old configurations, sensitive paths, or forgotten subdomains.
  • Social Media: Gather information about employees, technologies used, and company culture.
Example Shodan Query:
port:8080 country:"US" product:"nginx"

Active Reconnaissance

Directly interacting with the target network to gather more detailed information. This carries a higher risk of detection.

1. Ping Sweeps

Used to determine which hosts are active on a network. Tools like `nmap` can perform this.

nmap -sn 192.168.1.0/24

2. Port Scanning

Identify open ports and services running on target hosts.

Popular tools:
    - nmap (TCP SYN Scan, TCP Connect Scan, UDP Scan)
    - masscan (high-performance port scanner)

Example nmap scan:
nmap -sV -p- 192.168.1.100

3. Vulnerability Scanning

Identify known vulnerabilities in services and applications.

Tools:
    - Nessus
    - OpenVAS
    - Nikto (web server scanner)

4. OS Fingerprinting

Determine the operating system of a target host.

nmap -O 192.168.1.100

5. Enumeration (SMB, SNMP, SMTP, etc.)

Extracting more specific information about network resources, users, and shares.

Tools for SMB enumeration:
    - enum4linux
    - smbclient

Important Considerations

  • Stealth vs. Speed: Balance the need for secrecy with the efficiency of information gathering.
  • Legality & Ethics: Always obtain proper authorization before conducting active reconnaissance on systems you do not own or have explicit permission to test.
  • Documentation: Keep detailed records of all findings and methods used.
